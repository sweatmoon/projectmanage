시스템 개요

감리 공수관리 시스템이란?

감리 공수관리 시스템은 감리 사업·제안 프로젝트의 인력 배정, 일정 관리, 리스크 분석을 하나의 웹 인터페이스에서 통합 처리하는 시스템입니다. 인력의 프로젝트 간 중복 배정을 실시간으로 감지하고, 시뮬레이션을 통해 최적의 인력 조합을 제안합니다.

📁

프로젝트 & 단계 관리

감리 사업 및 제안 프로젝트의 단계(Phase)별 일정과 투입 인력을 등록·관리합니다.

👥

인력 정보 관리

감리원 등급, 소속 회사, 지방출장 가능 여부 등 인력 정보를 등록하고 검색합니다.

📅

인력별 일정 (월별 캘린더)

월별 캘린더 뷰로 인력별 투입 현황과 공수(MD)를 한눈에 파악합니다.

📊

사업별 일정 (Gantt)

사업 단위 Gantt 차트로 전체 일정과 인력 배치를 시각적으로 관리합니다.

🛡️

제안 리스크 분석

제안 사업의 인력이 기존 감리 사업과 일정 중복되는지 분석하고 교체 시뮬레이션을 제공합니다.

📈

프로젝트 목록 화면 기능

  • 전체 프로젝트를 사업(A) / 제안(P) / 수주(W) 구분으로 표시
  • 프로젝트명, 기관, 기간, 투입 인원 수 한눈에 확인

프로젝트 상세 화면

프로젝트를 클릭하면 단계(Phase)와 투입 인력(Staffing)을 관리할 수 있는 상세 화면으로 이동합니다.

단계(Phase) 관리

프로젝트를 단계로 나눠 시작일·종료일을 설정합니다. (예: 1단계, 2단계, 전체)

투입 인력(Staffing) 배정

각 단계에 감리원을 배정하고 공수(MD)를 입력합니다. 분야(field), 카테고리도 설정합니다.

실제 투입자(Hat) 지정

계획 인력과 실제 투입 인력이 다를 때 🎩 Hat 기능으로 실제 투입자를 별도 지정합니다.

일정 이동

단계 시작일·종료일을 수정하면 MD가 자동 재계산됩니다. 공수 초과 시 확인 다이얼로그가 표시됩니다.

⚠️
단계 일정을 변경할 때 기존 공수(MD)가 새 영업일 수를 초과하면, MD 자동 축소 또는 유지를 선택하는 다이얼로그가 나타납니다.

사업별 일정 탭은 사업(행) × 날짜(열) 형태의 Gantt 차트입니다. 각 사업의 단계(Phase)를 막대로 표시하고, 인력별 투입 현황을 확인·편집합니다.

주요 기능

📊

Gantt 막대 표시

각 사업의 단계(Phase)를 색상 막대로 표시. 사업(A) = 파란색, 제안(P) = 노란색.

✏️

인라인 인력 편집

막대를 클릭하면 해당 Phase의 투입 인력 목록이 펼쳐집니다. 인력 교체, MD 수정, 삭제가 가능합니다.

📅

일정 드래그 이동

막대를 드래그하여 Phase 시작일을 이동합니다. 날짜 변경 시 MD 재계산 다이얼로그가 표시됩니다.

🔍

필터 및 검색

사업 구분(전체/사업/제안), 기간 범위, 기관명으로 필터링합니다.

Excel 내보내기

인력 선택 다이얼로그에서 각 인력의 투입 가능 일수(완전가용/부분가용/투입불가)를 색상으로 구분합니다.

ℹ️
인력 가용성 색상
🟢 완전 가용 — 해당 Phase 전 기간 투입 가능
🟡 부분 가용 — 일부 기간 다른 사업과 겹침

리포트 — 통계 분석

단계별·구분별·인력별 공수 통계를 조회합니다

리포트 탭은 선택한 프로젝트의 투입 공수(MD) 통계를 다양한 기준으로 집계하여 표와 차트로 제공합니다.

📋

단계별 총 MD

1단계, 2단계 등 Phase별로 총 투입 공수를 합산하여 표시합니다.

🏷️

구분별 총 MD

단계감리팀, 핵심기술, 필수기술 등 투입 구분별로 공수를 집계합니다.

👤

인력별 총 MD (Top)

투입 공수가 많은 인력 순으로 상위 목록을 표시합니다.

⚠️
리포트를 보려면 먼저 프로젝트를 선택해야 합니다. 프로젝트가 없거나 선택되지 않으면 빈 화면이 표시됩니다.
🛡️

제안 리스크 — 일정 중복 분석

제안 사업 인력의 일정 중복을 분석하고 교체 시뮬레이션을 제공합니다

제안 리스크 탭은 제안 사업에 배정된 인력이 기존 감리 사업 또는 다른 제안 사업과 일정이 겹치는지 자동으로 분석합니다. 중복이 발견된 인력에 대해 교체 추천과 시뮬레이션을 통해 리스크를 최소화하는 방안을 제시합니다.

⚠️ 시뮬레이션은 실제 DB에 반영되지 않습니다. 가상의 교체 조합으로 리스크를 미리 계산하는 read-only 기능입니다.

리스크 유형

유형심각도설명
schedule_conflict 위험 감리 사업과 일정이 겹치는 인력
chief_overload 주의 총괄감리원이 3명 이상 배정된 경우
chief_role_conflict 위험 총괄감리원이 다른 사업과 날짜가 겹치는 경우
org_duplicate 주의 같은 기관에서 일정 기간 내 중복 배정

화면 구성 — 3가지 탭

① 최적화 추천

AI가 분석한 최적 인력 교체 조합 및 일정 이동 추천을 카드 형태로 제시합니다. 이 조합으로 시뮬레이션 버튼으로 바로 적용 가능합니다.

② 직접 시뮬레이션

인력 교체·일정 이동을 수동으로 설정하고, 실시간 자동 계산으로 리스크 변화를 확인합니다.

③ 텍스트 출력

시뮬레이션 결과를 복사 가능한 텍스트 형태로 출력합니다. 보고서 작성 시 활용합니다.

일정 이동 기능

  • 각 단계(Phase)의 시작일·종료일을 수정하면 즉시 재시뮬레이션
  • 날짜 수정 시 해당 행이 오렌지색 테두리로 강조됨
  • 초기화 버튼으로 개별 단계 원복 가능
  • 최적화 추천의 일정 이동 옵션을 클릭하면 자동 적용
💡
시뮬레이션 데이터 흐름
모든 시뮬레이션은 DB에 write하지 않습니다. 실제 일정 DB를 read-only로 조회한 뒤, 가상의 교체·이동 객체를 메모리에서 계산하여 결과를 반환합니다.
결과는 브라우저 상태(React state)에만 저장되며, 새로고침 시 사라집니다.
⚠️
writer 권한 전용: writer 역할을 가진 사용자는 제안 리스크 탭만 접근할 수 있습니다. 시뮬레이션 조회는 가능하지만 실제 데이터 수정 권한은 없습니다.
⚙️

관리자 페이지

admin 권한 전용 — 사용자 및 시스템 관리

관리자 페이지는 admin 역할만 접근할 수 있습니다. 상단 우측 프로필 아이콘 → 관리자 페이지 메뉴를 통해 진입합니다.

관리자 페이지 탭 구성

탭기능
📊 시스템 통계 전체 사용자 수, 오늘 로그인 수, 프로젝트/인력 수 등 시스템 현황 통계
📋 접근 로그 사용자별 로그인 기록, IP, 접근 시간 조회
👥 사용자 관리 등록된 모든 사용자 목록, 역할(role) 변경, 계정 삭제
🔍 감사 로그 데이터 변경 이력 (누가, 언제, 무엇을 변경했는지) 조회
🔔 권한 신청 접근 권한 신청 목록 조회, 승인/거부 처리. 미처리 건수가 배지로 표시됨

역할 변경 방법

  1. 1
    관리자 페이지 → 사용자 관리 탭 이동
  2. 2
    변경할 사용자 행의 역할 드롭다운 클릭
  3. 3
    원하는 역할(admin/leader/user/viewer/writer/audit_viewer) 선택
  4. 4
    즉시 반영됩니다 (해당 사용자 재로그인 시 적용)

권한 신청 승인 방법

  1. 1
    관리자 페이지 → 권한 신청 탭 (미처리 건 수 배지 표시)
  2. 2
    신청 사용자의 정보 확인 (이름, 이메일, 소속, 신청 사유)
  3. 3
    승인 버튼 클릭 → 승인 역할 선택 (admin/leader/user/viewer/writer)
  4. 4
    승인 완료 → 해당 사용자에게 접근 권한 부여

자주 묻는 질문 (FAQ)

Q. 로그인이 안 됩니다.

Google SSO 로그인을 사용합니다. 회사 Google 계정으로 로그인 후 권한 신청이 필요합니다. 관리자 승인 전에는 접근이 제한됩니다.

Q. 시뮬레이션 결과가 실제 DB에 저장되나요?

아닙니다. 제안 리스크의 시뮬레이션은 완전히 DB와 분리된 read-only 기능입니다. 결과는 브라우저 메모리에만 존재하며 새로고침 시 사라집니다.

Q. 인력별 일정에서 변경사항이 저장이 안 됩니다.

인력별 일정 탭에서는 상단 저장 버튼을 눌러야 DB에 반영됩니다. 탭을 이동하면 미저장 변경사항이 사라지므로 주의하세요.

Q. writer 권한인데 다른 탭이 안 보입니다.

writer 권한은 제안 리스크 탭만 접근 가능합니다. 다른 탭을 사용하려면 관리자에게 역할 변경을 요청하세요.

Q. 지방불가 표시는 어디서 설정하나요?

인력 탭에서 해당 인력 카드를 클릭 → 상세 화면의 지방 출장 항목에서 가능/불가 설정합니다.

Q. 🎩 Hat 기능은 무엇인가요?

계획 단계에서 배정된 인력과 실제 투입 인력이 다를 경우, Hat 기능으로 실제 투입자를 별도 지정할 수 있습니다. 인력별 일정 및 사업별 일정에서 🎩 버튼을 클릭해 설정합니다.

# Project Summary
This project is a comprehensive web application designed to enhance project management through an innovative scheduling interface. It allows users to effectively manage projects, phases, staffing, and personnel using a spreadsheet-like calendar grid system, promoting team collaboration and workflow efficiency. The application features a no-code approach for CRUD operations on project-related data. Recent updates include a redesigned `ScheduleTab` that supports a Gantt-chart style layout, introduces a unique badge system for displaying project phases, and addresses user feedback on badge visibility and assignment management. Additionally, improvements have been made to handle duplicate calendar entries effectively.

# Project Module Description
- **Backend**: Manages data handling, authentication, and CRUD operations for projects, phases, staffing, and users. Includes a `CalendarEntries` model for tracking daily staffing statuses and a new endpoint for cleaning up duplicate entries.
- **Frontend**: Offers a user-friendly interface for managing projects and staffing. The `ScheduleTab` component now displays personnel assignments in a Gantt-chart style, utilizes badges for project phases, and ensures unique badge representation without duplication. Enhancements have been made to the auto-fill logic to prevent duplicate entries.

# File Description Inventory
- **backend/**: Contains server-side code, including models, routes, and services. The `calendar_bulk.py` file has been updated to handle duplicate entries and includes a new endpoint for cleaning up duplicates.
- **frontend/**: Contains client-side application components, pages, and assets. The `ScheduleTab.tsx` component has been redesigned to support a Gantt-chart style layout, unique badge representation, and improved assignment management, including enhancements to prevent duplicate entries during auto-fill.

# Technology Stack
- **Backend**: Python, FastAPI, SQLAlchemy
- **Frontend**: React, TypeScript, Vite, Tailwind CSS
- **Database**: JSON for mock data, with plans for relational database integration.

# Usage
1. **Install Dependencies**:
   - For backend: Run `pip install -r requirements.txt`.
   - For frontend: Run `pnpm install`.
2. **Build the Application**:
   - Navigate to the frontend directory and run `pnpm run build`.
3. **Run the Application**:
   - Start the backend server using the command that runs your FastAPI application.
   - Start the frontend development server using `pnpm run dev`.
